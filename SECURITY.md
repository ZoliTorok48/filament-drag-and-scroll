# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this package, please report it privately so it can be addressed before public disclosure.

**Do not** open a public GitHub issue for security vulnerabilities.

Send reports to:

**Email:** [torokzoltan0348@gmail.com](mailto:torokzoltan0348@gmail.com)

Include as much detail as you can:

- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Affected versions
- Any proof-of-concept code or screenshots (if applicable)

## Response Timeline

- **Acknowledgment:** within 7 days of receiving your report
- **Status update:** within 14 days, with an initial assessment and planned fix timeline
- **Resolution:** as soon as a fix is ready, we will release a patched version and publish a security advisory when appropriate

## Supported Versions

Security fixes are provided for the latest stable release. Older releases may receive fixes at the maintainer's discretion depending on severity and feasibility.

## Recognition

We appreciate responsible disclosure. With your permission, we are happy to acknowledge your contribution in the release notes or advisory.
